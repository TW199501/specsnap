## v0.0.3 主題：讓 AI 同時拿到「視覺 + 結構」

### Why now

今日在 antares2 整合 `@tw199501/specsnap-core@0.0.2` 實戰後的觀察（整理自 `feedback` + conversation）：

- 純 Markdown 的 classList / color / size 欄位已經能解 **80%** 的 UI 診斷（今天 footer / misc-name / toggle / query-info 4 個 bug 都只靠 text 定位）
- 剩下 **20%** 是「格子排不齊」「邊界有怪影」之類**視覺現象**，光講不夠，得看圖
- 使用者今天貼截圖的次數：0（因為 SpecSnap 夠用）。但 `gap` section 的出現讓使用者第一次意識到「原來結構化輸出可以讓 AI 不用問就懂」— 這股動能要延續下去
- Screenshot export 在 `memory/project_specsnap_screenshot_gap.md` 是 **2026-04-19 已 flagged 的優先項**

### Scope

**Task 1: `toAnnotatedPNG(session, options?)` API**

新檔 `packages/core/src/to-annotated-png.ts`（core 擴 API，不是 playground-only）：

```ts
export interface AnnotatedPngOptions {
  /** Include numbered badges from session.frames[].index. Default: true */
  badges?: boolean;
  /** Include orange gap lines + px labels from session.gaps. Default: true */
  gaps?: boolean;
  /** Include red dashed outline on the parent of the last frame. Default: true */
  parentOutline?: boolean;
  /** Output format. Default: 'png' */
  format?: 'png' | 'jpeg';
  /** JPEG quality 0-1. Ignored for PNG. Default: 0.92 */
  quality?: number;
  /** Pixel ratio for retina. Default: session.viewport.devicePixelRatio */
  pixelRatio?: number;
}

export async function toAnnotatedPNG (
  session: Session,
  options?: AnnotatedPngOptions
): Promise<Blob>;
```

**實作路徑**：
- 抽 `apps/playground/visualizer.ts` 的 SVG 繪製邏輯到 `packages/core/src/annotate.ts`（DOM-independent，只要 Session + bounds-by-index map）
- 用 `dom-to-image-more`（runtime dep，首次 `toAnnotatedPNG` 才 dynamic import，不影響 tree-shake）
- 流程：
  1. 在 session 涵蓋的元素 bounding box 計算 minimum bounding rect
  2. 臨時在 document 建立一個 overlay SVG（position: absolute, top/left 對齊 bbox）
  3. `dom-to-image-more.toBlob(document.body, { filter: node => inRect(node, bbox) })` 截取該區域
  4. 移除臨時 SVG
  5. 回傳 Blob

**Task 2: `toMarkdown` 選項支援嵌入截圖**

擴 `SerializeOptions`：

```ts
export interface SerializeOptions {
  lexiconOverride?: Record<string, string>;
  pretty?: boolean;
  /** Embed annotated PNG as base64 data URL at the top of frame 1's MD. Default: false */
  embedScreenshot?: boolean;
}
```

開啟時，`toMarkdown` 呼叫 `toAnnotatedPNG`，把結果 base64 嵌入 YAML frontmatter 後、` # Frame 1 · ...` 之前：

```markdown
---
frame: 1 of 2
...
---

![captured](data:image/png;base64,iVBOR...)

# Frame 1 · button#save
...
```

這樣使用者貼 Markdown 給 AI，AI 同時拿到文字 + 圖 — 真正做到「pencil-like session replay」。

**Task 3: Playground 加「下載截圖」按鈕**

按鈕 state：`idle / generating / ready`。生成後 `<a download="specsnap-{sessionId}.png">` 觸發瀏覽器下載。

**Task 4: antares2 TheSpecSnapInspector 加「複製成圖」選項**

第三顆 tab：`Markdown | JSON | **圖片**`。點「複製」時：
- Markdown tab → 複製 markdown
- JSON tab → 複製 json
- 圖片 tab → 預覽 annotated PNG + 複製到剪貼簿（`navigator.clipboard.write([new ClipboardItem({...})])`）

### Source-location hint（附加任務）

不算主項，但可搭：**Vite 開發模式下把 `__v_source` 屬性注入每個 DOM 節點**，指向源檔。這需要寫一個 Vite plugin `@tw199501/specsnap-vite-plugin` 插入 Vue SFC 編譯 pipeline。範圍大，**列 0.0.5 候選**，本輪不做。

但 core 的 `captureElement` 可以先**讀**該屬性（如果存在）寫進 `ElementIdentity.source?: string`。無害擴充，為未來 plugin 鋪路。

### Ship criteria

- [ ] `toAnnotatedPNG` 實測生出 PNG Blob，尺寸合理（不是 1920×1080 全螢幕）
- [ ] `toMarkdown({ embedScreenshot: true })` 產生的 MD 貼進 Claude，截圖被正確顯示
- [ ] Playground 截圖 + Markdown 兩者視覺一致（badges/gaps 對齊）
- [ ] `@tw199501/specsnap-core@0.0.3` 上 npm
- [ ] 檔案大小：~30-40 KB tarball（加 dom-to-image-more 的包）
- [ ] 測試：現有 50 tests 全綠 + 新增 `annotate.test.ts` + `to-annotated-png.test.ts`（後者可能只能在 playwright 環境跑，happy-dom 不支援 canvas）

## v0.0.4 主題：schema 深度 + AI reverse-lookup

0.0.3 把「視覺」補齊後，0.0.4 補「程式語意」。3 個候選選 2-3 個做。

### Candidate 1: Pseudo-state capture

**Why**：每次 antares2 改按鈕樣式都會想「disabled 看起來怎樣？hover 呢？」。這資訊在 DOM 存在但 `getComputedStyle` 只抓 current state。

**How**：`Frame.pseudoStates?: { hover?, focus?, active?, disabled? }`，用 `getComputedStyle(el, ':hover')` 之類的（注意：多數瀏覽器不支援直接 query pseudo-class computed style，要用 inspector API 或 `window.getMatchedCSSRules` — 舊 API 不穩）。

**Fallback**：若 runtime 取不到，接受 `undefined`（欄位是 optional）。文件標明「若瀏覽器支援則提供」。

### Candidate 2: i18n key reverse-lookup

**Why**：今天 antares2 的「資料」按鈕文字是 `t('general.data')`，SpecSnap 只能寫 `button[text="資料"]`，我得再 grep i18n 才能找到 key。如果直接輸出 `t(general.data)` 就省一步。

**How**：兩條路
- (a) Vue instance hook：用 `vnode.__v_i18n_key` 之類的內部屬性（脆弱，跨框架不通用）
- (b) 在 DOM 裡加 `data-i18n-key="general.data"` 屬性（需要 build-time 注入，屬於 Vite plugin 範疇 → 延後到 0.0.5）

**Decision**：0.0.4 只做 core 的「**若存在 `data-i18n-key` 就讀入**」支援。注入工具等 0.0.5。

### Candidate 3: Recursive children dump

**Why**：0.0.1 vision 的 P5 principle。使用者今天抓 `.workspace-query-info` 時是 **5 個子元素各抓一次**。若父元素抓一次就能帶出「有意義的」子元素，UX 變更好。

**How**：`CaptureOptions.recursive?: { depth?: number; filter?: (el) => boolean }`。預設 filter = 「有可見文字 / 有 interactive role / 有非繼承樣式」— 這個 filter 本身是個 design decision，需花時間定義。

**Scope warning**：這會讓 Session JSON 暴增，可能要加 `maxFrames` 限制。

### 推薦組合：1 + 2 + 3（選 2）

- **必做 1**（pseudo-states）：實戰需求最高
- **必做 2**（`data-i18n-key` 讀取）：低成本、為 0.0.5 鋪路
- **候選 3**（recursive）：若前兩者提早完成就做，否則延 0.0.5

## 累積 insight（給自己的筆記）

從 antares2 整合 + 6 輪 UI 修復學到的事：

1. **SpecSnap 真的有用，但 AI 還沒在利用全部欄位** — `domPath` 我幾乎沒用過（nth-of-type 對 Vue 組件定位無意義）、`viewportRelative` 百分比也沒用過。下一版要考慮**精簡 schema** 讓 AI 聚焦在有用的欄位（`classList` / `rect` / `color` / 等），或把不常用欄位移到 `extra?:` 可選區
2. **`name` 欄位的 `text=` 後綴在 i18n 專案會誤導** — 應該加個 optional `textKey?: string` 欄位，由 build-time plugin 填
3. **Gaps 的設計超成功** — `Frame N → Frame M: Npx horizontal (水平間距)` 這種自然語言輸出讓 AI 一眼看懂關係。0.0.3 的 annotate PNG 也要用同樣視覺語言保持一致性
4. **Playground 是最好的 bug detector** — 今天 antares2 的每個 UI bug 都能被 SpecSnap 抓到，代表 capture 足夠細緻。但也意味著 playground 的 visualizer 邏輯需要穩定化（moved to core in 0.0.3）
5. **npm 發佈流程已熟** — 0.0.2 shipping smooth，0.0.3 可以不用太緊張
